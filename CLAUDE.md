# 文件创建规则
- **禁止**：未经允许，不得创建任何 .md 文件

- **例外**：`.claude/.execution-plan.md` 是 Codex 执行流程的计划文件，允许直接创建和更新

# Codex 协作流程

## 流程一：Codex 审核（用户说"codex审核"触发）
- Claude 写完代码后，用户说"codex审核"
- Stop hook 自动调用 Codex 审核代码改动
- 审核结果展示后，用户决定是否按建议修改

## 流程二：Codex 执行（用户说"codex执行"触发）
- **关键**：用户提出任务后，Claude 必须先将详细实施计划写入 `.claude/.execution-plan.md`
- 计划写好后告知用户，用户说"codex执行"触发 Codex 按计划写代码
- 计划格式要求：
  - 明确列出每个要操作的文件路径（用 `路径: \`xxx\`` 格式）
  - 详细到方法签名、类名、具体实现逻辑
  - Codex 只会操作计划中提到的文件，不会触碰其他文件
- 执行完后用户确认是否保留改动，不同意则回滚
