## 📊 功能特性

### 1. **组织管理** 📋
- 添加/删除组织名称（公司、品牌等）
- 示例：Dell, HP, Lenovo, 小米, OPPO, vivo

### 2. **地区管理** 🌍
- 添加/删除地理位置名称
- 示例：华东, 华南, 华北, 美国, 中国

### 3. **表别名** 📊
- 业务名称到数据库表名的映射
- 示例：产品 → statshipmentboxai

### 4. **列别名** 📝
- 业务名称到数据库列名的映射
- 示例：销售额 → Volume

### 5. **列值域** 🎯
- 管理每列的有效值集合
- 示例：PRODUCT_BRAND → [Dell, HP, Lenovo, ...]

### 6. **产品系列** 📦
- 产品系列到型号的映射
- 示例：Ideapad → [Slim 5, Gaming 3, ...]

### 7. **业务术语** 💼
- 术语缩写到完整含义的映射
- 示例：SS → Listing, CTO → Configure To Order

### 8. **财年规则** 📅
- 设置财年起始月份和日期
- 管理财年映射

## 📝 使用步骤

### 添加"小米"等品牌

#### 1. 访问页面
```
http://localhost:3000/knowledge-graph
```

#### 2. 添加到组织列表
1. 点击 **"组织管理"** 标签
2. 在输入框输入："小米"
3. 点击 **"添加"** 按钮
4. 重复添加："Xiaomi", "红米", "Redmi", "OPPO", "vivo"

#### 3. 添加到产品品牌值域
1. 点击 **"列值域"** 标签
2. 在左侧找到 **"PRODUCT_BRAND"**，点击选中
3. 在右侧输入框添加："小米"
4. 继续添加："Xiaomi", "红米", "Redmi", "OPPO", "vivo"

#### 4. 保存配置
1. 点击页面顶部的 **"保存配置"** 按钮
2. 系统会弹出确认框：**"配置已保存。是否立即刷新知识图谱？"**
3. 点击 **"确定"** 自动刷新知识图谱

#### 5. 验证
- 在聊天页面输入："小米的销售额"
- 系统应该能正确识别并生成SQL

## ⚙️ 操作按钮说明

### 1. 保存配置
- **功能**: 保存当前修改到服务器
- **状态**: 只有修改后才可用
- **注意**: ⚠️ 保存后需要刷新知识图谱才能生效

### 2. 刷新图谱
- **功能**: 将配置加载到内存
- **时机**: 保存配置后必须执行
- **效果**: 新配置立即生效

### 3. 创建备份
- **功能**: 备份当前配置
- **建议**: 修改前先备份
- **位置**: 服务器 `backend/backups/` 目录

### 4. 重新加载
- **功能**: 从服务器重新加载配置
- **用途**: 放弃未保存的修改

---

## ⚠️ 重要注意事项

### 1. **保存后必须刷新**

```
保存配置 → 确认刷新 → 配置生效 ✅
```

**错误做法**:
```
保存配置 → 不刷新 → 配置未生效 ❌
```

### 2. **配置文件位置**

**后端配置文件**（源文件）:
```
backend/src/main/resources/entity-graph/business-entities.json
```

**编译后文件**（运行时读取）:
```
backend/target/classes/entity-graph/business-entities.json
```

### 3. **已知问题与限制**

**之前的保存 bug 已修复** ✅：
- 现在后端能够自动同步更新源代码（`src/main/resources`）
- 即使 Maven 重新编译，配置也不会丢失
- 修复了 JSON 格式化问题

**注意事项**：
- 如果你在 IDE 中同时编辑了 `business-entities.json`，请确保存盘，以免覆盖。
- 建议定期使用"创建备份"功能。
